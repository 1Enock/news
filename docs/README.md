# Documentation Guidelines

Keep all lab documents and solutions under this folder. Use the following structure:

- `docs/specs/` — experiment specs and protocols
- `docs/solutions/` — reference solutions and walkthroughs
- `docs/templates/` — document templates (lab report, checklist)

Guidelines:
- Prefer Markdown (`.md`).
- Include a short summary, prerequisites, steps, and expected results.
- Link example code from `examples/` when relevant.
